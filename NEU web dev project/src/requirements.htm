Use Cases

Login Page
Description: Pre-registered users are allowed to login and use their premium features and functionalities after successful log in. Precondition: User must be registered user to get successful login. Steps: Go to the museum websites home page Click on the Login button and enter the registered email address and the corresponding password Click on Log in button and the user logs in as museum member. Post condition: On successful log in, the user should be redirected to their respective profile page. If log in was unsuccessful then user will be shown message error message to re-login or use forgot username / password feature to retrieve user name / password.
Museum Locator -Maps Page
Description: User would like to locate the museum he wants to visit Precondition: User should know the museum names listed Steps: one or more of the following User selects a museum from the list of museums to be located The selected museums location is displayed on the Map to the right Post condition: The location of the museum is shown on the map Alternate paths (scenarios): User would like to locate multiple museums of the group on the same map by just clicking on different museum link.
Search Page
Description: User would like to get information about artifacts at the museum and also be able to view some images of the artifacts, they can search based on artists name, keyword or exhibits for display. Precondition: User should know the type of artifact that he will be interested to know about. Steps: User picks his search criteria from the radio buttons to specify search criteria User puts in the keyword and he should see a set of results related to his search Post condition: The corresponding artifacts description, date and images are displayed. Artifact images and related images are displayed in a modal window
Tickets Purchase-Cart Page
Description: User would like to pick a date favorable for him and based on the type of ticket he can purchase like a student, senior citizen or general ticket. Precondition: the user should know the type of ticket that he is eligible for Steps: User selects a date from the calendar based on their requirements User choses the type of ticket or offer based on his ticket type Post condition: The corresponding ticket details will be displayed like the details or identification documents that the user should furnish to be eligible for the ticket Alternate paths (scenarios): User should not be able to purchase tickets on days there are planned maintenance and issues.
Feedback Page
Description: User would like to give feedback about the museum experience and also be able to view other users feedbacks as well. Precondition: User should have registered and must login to provide feedback and view others comments. Steps: User logs in with username and password User must provide a name and the feedback and click submit User can also just view other feedback Post condition: The users artifacts are submitted along with name, timestamp and the message.
